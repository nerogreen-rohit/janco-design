# Task P4-07: Configure Portal Permissions

**Phase:** 4 — Customer Portal  
**Story:** As a developer, I need to configure permissions so customers can only see their own data  
**Priority:** 🔴 Critical  
**Estimated effort:** 1–2 hours  
**Depends on:** P4-01 through P4-06 (all portal components configured), Phase 1 (guest links operational)

---

## Step-by-Step Guide

### Step 1 — Understand the Permission Model

The customer portal relies on **item-level and folder-level guest sharing** combined with **filtered views**. Customers never receive site-level or list-level access.

#### Customer CAN Access

| Resource | Access Level | How Enforced |
|---|---|---|
| Their Fact-Find item | Edit | Guest-shared direct item link (generated by Phase 1 Flow 4) |
| Document upload folders 01–04 | Contribute | Guest-shared folder links (generated by Phase 1 Flow 5) |
| Messages filtered to their CaseID | Read / Contribute | Guest-shared filtered view link |
| Progress page | Read only | SharePoint page shared with customer (P4-01) |

#### Customer CANNOT Access

| Resource | How Enforced |
|---|---|
| Other customers' Fact-Find items | Direct item link — customer only has link to their own item |
| Other customers' case data | No list-level sharing; filtered views per CaseID |
| Internal staff notes (Direction = "Internal") | Filtered view excludes Internal messages |
| Internal stage names | Portal shows `CustomerFriendlyName` only |
| Commission records | No sharing applied to Commissions list |
| Document folders 05–13 | Guest sharing applied to folders 01–04 only |
| Other cases' folders | Guest links are case-specific |
| Staff App (Power App) | No Power Apps license for customers |

### Step 2 — Verify Fact-Find Item Isolation

1. Create two test cases:
   - **Case A:** `BTL-2026-TEST-A` for Customer A (`customera@test.com`)
   - **Case B:** `BTL-2026-TEST-B` for Customer B (`customerb@test.com`)
2. Get the fact-find guest link for Customer A
3. Open in an InPrivate/Incognito window and sign in as Customer A
4. **Verify Customer A CAN:**
   - Open their fact-find edit form
   - Edit and save their fact-find
5. **Verify Customer A CANNOT:**
   - Browse the full Fact-Find Responses list (`https://[tenant].sharepoint.com/sites/lending/Lists/FactFindResponses`) → **Access Denied**
   - See Customer B's fact-find data
   - Navigate to the Cases list → **Access Denied**
   - Navigate to any other SharePoint list → **Access Denied**

### Step 3 — Verify Folder Permission Isolation

1. Get the guest link for Customer A's `01-Identity` folder
2. Open in InPrivate/Incognito as Customer A
3. **Verify Customer A CAN:**
   - Upload files to `01-Identity`
   - View files in `01-Identity`
   - Upload files to `02-Income`, `03-Property`, `04-Bank-Statements` (via their respective links)
4. **Verify Customer A CANNOT:**
   - Navigate to the parent folder (`Case Documents/BTL-2026-TEST-A/`)
   - Access folder `05-Fact-Find` directly
   - Access folders `06` through `13`
   - Access any of Customer B's folders

### Step 4 — Verify Messages Isolation

1. Create test messages for both cases:
   - Case A: Advisor → Customer message + Internal note
   - Case B: Advisor → Customer message
2. Open the Customer Messages view filtered to Case A's CaseID
3. **Verify Customer A CAN:**
   - See "Advisor → Customer" messages for Case A
   - See "Customer → Advisor" messages for Case A
   - Create a reply (new message item)
4. **Verify Customer A CANNOT:**
   - See "Internal" direction messages
   - See any messages for Case B
   - Browse the unfiltered Messages list

### Step 5 — Verify Progress Page Isolation

1. Open Customer A's portal page as Customer A
2. **Verify Customer A CAN:**
   - View the progress tracker for their case
   - See CustomerFriendlyName values (not internal stage names)
3. **Verify Customer A CANNOT:**
   - See Stage 4 (Product Research) — it should be hidden
   - Navigate to Customer B's portal page
   - Access Site Contents or Site Settings
   - See the Stage Configuration list directly

### Step 6 — Verify Document Request Isolation

1. Open Customer A's portal page
2. **Verify the "What You Need To Do" panel:**
   - Shows only Document Requests for Case A
   - Does NOT show Document Requests for Case B
   - Does NOT show Received/Approved items (only Requested/Rejected)

## Troubleshooting

| Issue | Resolution |
|---|---|
| Customer can browse full list | The sharing was applied at list level, not item level — revoke and re-share at item/folder level |
| Customer sees other cases' messages | CaseID filter is missing on the embedded view; add filter to the web part or view |
| Customer can access folder 05+ | Sharing was applied to the parent folder — revoke parent sharing; share only 01–04 individually |
| Customer sees internal stage names | Portal page is using `StageName` instead of `CustomerFriendlyName` — update the view or template |
| Customer can navigate to Site Contents | Page was shared with site-level access — revoke and re-share at page level only |
| Guest link expired | Check guest link expiry settings in IT Admin config (P4-08); regenerate links |
