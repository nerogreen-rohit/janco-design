# Task P4-06: Configure Customer Document Upload & View

**Phase:** 4 — Customer Portal  
**Story:** As a customer, I want to view and upload my documents so that I can provide everything the advisor needs  
**Priority:** 🟡 High  
**Estimated effort:** 1 hour  
**Depends on:** P4-01 (portal page created), Phase 1 Flow 5 (folder guest share links generated)

---

## Step-by-Step Guide

### Step 1 — Understand the Document Folder Structure

Phase 1 Flow 5 created 13 subfolders per case in the Case Documents library. Only folders 01–04 are shared with the customer:

| Folder | Name | Customer Access |
|---|---|---|
| 01 | Identity | ✅ Contribute (upload + view) |
| 02 | Income | ✅ Contribute (upload + view) |
| 03 | Property | ✅ Contribute (upload + view) |
| 04 | Bank-Statements | ✅ Contribute (upload + view) |
| 05–13 | *(staff-only folders)* | ❌ No access |

Guest share links for folders 01–04 were generated by Phase 1 Flow 5 and stored on the Cases list record.

### Step 2 — Locate the Folder Guest Links

1. Navigate to the **Specialist Lending Cases** list
2. Find the customer's case record (e.g. `BTL-2026-0001`)
3. Locate the stored folder guest links:
   - `Folder01GuestLink` → 01-Identity
   - `Folder02GuestLink` → 02-Income
   - `Folder03GuestLink` → 03-Property
   - `Folder04GuestLink` → 04-Bank-Statements
4. Copy each URL

> **Link format example:** `https://[tenant].sharepoint.com/:f:/s/lending/[unique-share-id]`

### Step 3 — Add Document Links to the Portal Page

1. Open the customer's portal page (P4-01) in **Edit** mode
2. Navigate to the **LINKS** section (or the **WHAT YOU NEED TO DO** section)
3. Add a **Text** web part
4. Create the document upload links:

```
📁 Your Document Folders

Upload your documents to the appropriate folder below:

📂 Identity Documents (passport, driving licence)
📂 Income Documents (payslips, tax returns, SA302)
📂 Property Documents (EPC, lease, planning permission)
📂 Bank Statements (last 6 months)
```

5. Select each folder name → click the **Link** icon (🔗) → paste the corresponding guest link
6. Ensure each link opens in a new tab

### Step 4 — Add a Quick Upload Button

For a simpler experience, add a single "Upload Documents" button:

1. Add a **Button** web part
2. Label: `📤 Upload Documents`
3. Link to a page with all four folder links, OR link to the primary folder (e.g. 01-Identity) as a starting point

> **Tip:** If using the HTML embed template (`imports/portal-page-template.html`), replace `{{Folder01Link}}` through `{{Folder04Link}}` with the actual guest links.

### Step 5 — Add a View of Already-Uploaded Documents (Optional)

To let customers see what they've already uploaded:

1. Add a **Document Library** web part to the portal page
2. Select the **Case Documents** library
3. Filter to the customer's case folder: `CaseID/01-Identity`, `CaseID/02-Income`, etc.

> **Note:** This requires the document library folders to be individually shared with the customer at Read level. Phase 1 Flow 5 already provides Contribute access to folders 01–04, so uploaded files should be visible.

**Alternative — Show document status via Document Requests list:**

1. Add a **List** web part
2. Select the **Document Requests** list
3. Create a view filtered by CaseID showing all statuses:

| DocumentName | Status | ReceivedDate |
|---|---|---|
| Passport | ✅ Received | 2026-02-10 |
| Payslips | ✅ Received | 2026-02-12 |
| Bank Statements | ⏳ Requested | — |
| EPC Certificate | ❌ Rejected | — |

4. Apply view formatting:

```json
{
  "$schema": "https://developer.microsoft.com/json-schemas/sp/v2/row-formatting.schema.json",
  "hideColumnHeader": false,
  "rowFormatter": {
    "elmType": "div",
    "style": {
      "display": "flex",
      "align-items": "center",
      "padding": "6px 12px",
      "border-bottom": "1px solid #f0f0f0"
    },
    "children": [
      {
        "elmType": "span",
        "style": {
          "font-size": "16px",
          "margin-right": "10px",
          "min-width": "24px"
        },
        "txtContent": "=if([$Status] == 'Received', '✅', if([$Status] == 'Approved', '✅', if([$Status] == 'Rejected', '❌', if([$Status] == 'Requested', '⏳', '○'))))"
      },
      {
        "elmType": "span",
        "style": {
          "font-size": "14px",
          "flex": "1"
        },
        "txtContent": "[$DocumentName]"
      },
      {
        "elmType": "span",
        "style": {

          "color": "=if([$Status] == 'Received', '#107c10', if([$Status] == 'Rejected', '#d13438', '#666666'))"
        },
        "txtContent": "[$Status]"
      }
    ]
  }
}
```

### Step 6 — Test Document Upload Flow

1. Open the portal page as a guest customer (InPrivate/Incognito window)
2. Click the `📂 Identity Documents` link
3. Verify the folder opens in SharePoint
4. Upload a test file (e.g. `test-passport.pdf`)
5. Verify:
   - File appears in the folder
   - Phase 3 Flow 8 triggers (creates/updates Document Request item as "Received", notifies case owner)
6. Repeat for folders 02, 03, and 04

### Step 7 — Test Access Restrictions

1. As the guest customer, attempt to navigate to the parent folder (`Case Documents/[CaseID]/`)
2. Verify: **Access Denied** or no navigation path available
3. Attempt to access folder `05-Fact-Find`
4. Verify: **Access Denied**
5. Attempt to access another case's folders
6. Verify: **Access Denied**

## Troubleshooting

| Issue | Resolution |
|---|---|
| Folder link shows "Access Denied" | Guest share link may have expired; regenerate via Phase 1 Flow 5 or manually re-share |
| Customer cannot upload files | Verify the share link grants Contribute (not Read) access |
| Customer can see parent folder | Check folder-level sharing is applied to subfolders, not the parent |
| Uploaded file not triggering Flow 8 | Check the Phase 3 flow trigger condition; verify the flow is enabled |
| Wrong folder opens | Verify the link mapping: Folder01 → 01-Identity, Folder02 → 02-Income, etc. |
| Document status view shows all cases | CaseID filter missing on the list web part; add the filter |
