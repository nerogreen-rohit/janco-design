# Task P4-03: Configure the "What You Need To Do" Action Panel

**Phase:** 4 — Customer Portal  
**Story:** As a customer, I want to see what documents are still needed ("What you need to do" panel) so that I can take action promptly  
**Priority:** 🔴 Critical  
**Estimated effort:** 1–2 hours  
**Depends on:** P4-01 (portal page created), Phase 3 (Document Requests list operational)

---

## Step-by-Step Guide

### Step 1 — Understand the Action Panel

The "What You Need To Do" panel shows the customer any outstanding items that require their attention. It pulls from the **Document Requests** list and shows only items for their case with an actionable status.

| Displayed when | Status value |
|---|---|
| Document initially requested | `Requested` |
| Document was rejected by advisor (re-upload needed) | `Rejected` |
| *(Not shown)* | `Received`, `Approved` |

### Step 2 — Create the Customer Action View on Document Requests

1. Navigate to the **Document Requests** list
2. Click **⚙️** → **List Settings** → **Views** → **Create view**
3. Name: `Customer Action Items`
4. Configure the view:
   - **Columns to show:**
     - `DocumentName`
     - `Description`
     - `DueDate`
     - `Status`
   - **Filter:**
     - `Status is equal to Requested` **OR** `Status is equal to Rejected`
   - **Sort:** `DueDate` ascending (most urgent first)
5. Click **OK**

> **Note:** The CaseID filter will be applied at the web part level on the portal page (Step 4) or via the view URL.

### Step 4 — Embed the Action Panel on the Portal Page

1. Open the customer's portal page (created in P4-01)
2. Navigate to the **WHAT YOU NEED TO DO** section
3. Add a **List** web part
4. Select the **Document Requests** list
5. Select the **Customer Action Items** view
6. Click **Edit web part** (pencil icon) and configure the filter:
   - **Filter:** `CaseID is equal to [the customer's CaseID]`

> **Alternative approach:** Create a separate view per customer case (e.g. `Action Items — BTL-2026-0001`) with the CaseID filter baked into the view definition. This avoids relying on web part filters.

## Troubleshooting

| Issue | Resolution |
|---|---|
| All document requests showing (not filtered) | Verify the CaseID filter on the web part matches exactly; check for trailing spaces |
| "Received" items still visible | Confirm the view filter uses OR logic: `Status = Requested OR Status = Rejected` (not AND) |
| View formatting not rendering | Ensure JSON is valid; test in a standalone view before embedding |
| Upload links not working | Verify the guest share links from Phase 1 Flow 5 are still active and correctly stored on the Cases list |
| DueDate showing incorrectly | Check date format in the JSON: `toLocaleDateString()` should render in the user's locale |
