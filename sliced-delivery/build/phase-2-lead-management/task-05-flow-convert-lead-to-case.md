# Task P2-05: Flow 2 — Convert Lead to Case

**Phase:** 2 — Lead Management  
**Story:** As a staff member, when I click [Convert to Case] on a qualified lead, the system should create a Case record pre-populated from the lead data and mark the lead as converted  
**Priority:** 🔴 Critical  
**Estimated effort:** 1–2 hours  
**Depends on:** Leads list (P2-01), Cases list (P1-01), Flows 3/4/5 from Phase 1

---

## Step-by-Step Guide

### Step 1 — Create the Flow

1. Go to **Power Automate** → **+ Create** → **Instant cloud flow**
2. Name: `Flow 2 — Convert Lead to Case`
3. Trigger: **For a selected item** (SharePoint) OR **Power Apps (V2)** — see notes below
4. If using SharePoint trigger:
   - Site: `https://[tenant].sharepoint.com/sites/lending`
   - List Name: `Leads`

> **Recommended approach:** Trigger this flow from Power Apps using `Power Automate` → `Run a flow` button in the Staff App, passing the Lead ID as a parameter.

### Step 2 — Get the Full Lead Record

1. Add **Get item** (SharePoint):
   - List: `Leads`
   - Id: `triggerOutputs()?['body/ID']` (or the ID parameter from Power Apps)

### Step 3 — Validate Lead Status

1. Add **Condition**:
   - `outputs('Get_item')?['body/LeadStatus/Value']` is equal to `Qualified`
2. If **No**: 
   - **Respond to a Power App** / **Terminate** with: "Lead must be Qualified to convert"
   - End flow

### Step 4 — Create the Case Record

1. If **Yes**, add **Create item** (SharePoint):
   - List: `Specialist Lending Cases`
   - Map the following fields from the Lead:

| Case Field | Lead Source |
|---|---|
| ProductType | `ProductInterest` |
| Applicant1FirstName | `FirstName` |
| Applicant1LastName | `LastName` |
| Applicant1Email | `Email` |
| Applicant1Phone | `Phone` |
| PropertyAddress | `PropertyAddress` |
| LoanAmountRequired | `LoanAmountRequired` |
| LeadID | `LeadID` |
| CaseOwner | `AssignedTo` |
| EnquiryDate | `utcNow()` |
| CaseStatus | `Active` |
| ReferrerID | `ReferrerID` (if populated) |

### Step 5 — Wait for Case Setup Flows

After the Case is created, Flows 3, 4, and 5 trigger automatically:
- Flow 3: Generates CaseID
- Flow 4: Creates Fact-Find item + guest link
- Flow 5: Creates folders + sends welcome email

Add a **Delay** of 60 seconds or **Do until** CaseID is populated on the new Case record.

### Step 6 — Update the Lead Record

1. Add **Get item** — re-read the Case to get the generated CaseID
2. Add **Update item** (SharePoint):
   - List: `Leads`
   - Id: original Lead ID
   - ConvertedToCaseID: `[generated CaseID]`
   - ConvertedDate: `utcNow()`
   - LeadStatus: `Converted`

### Step 7 — Return Result to Power App

If triggered from Power Apps:
1. Add **Respond to a Power App** with:
   - Success: `true`
   - CaseID: `[generated CaseID]`
   - Message: `"Lead converted to case {CaseID} successfully."`

### Step 8 — Test

1. Create a Lead, set status to "Qualified"
2. Trigger the Convert flow (from the Staff App or SharePoint)
3. Verify:
   - [ ] Case record created with data from the lead
   - [ ] CaseID generated by Flow 3
   - [ ] Fact-Find item created by Flow 4
   - [ ] Folders created by Flow 5
   - [ ] Welcome email sent to customer
   - [ ] Lead updated: ConvertedToCaseID set, ConvertedDate set, LeadStatus = "Converted"
   - [ ] Lead no longer appears in Active Leads view
4. Try converting a Lead with status "New" → should fail with validation error

## Troubleshooting

| Issue | Resolution |
|---|---|
| Case created but CaseID not generated | Flow 3 may have failed — check flow run history |
| Welcome email not sent | Flow 5 may be waiting for CaseID — check the Do Until timeout |
| Lead not updated with CaseID | Check the Delay or Do Until step — the Case creation flows need time to complete |
| Convert fails with "Lead must be Qualified" | Lead status is not "Qualified" — verify the condition check |
