## 5.5 Case Stage History List

**Purpose:** Immutable audit log of every stage transition for each case.

> **Design note:** This list stores text snapshots (not lookups) to preserve audit integrity. Even if the Stage Configuration list is later modified or a stage is renamed, the history record shows exactly what the stage was called at the time of the transition.

| Column | Type | Notes |
|---|---|---|
| Title | Single line | Set to CaseID + Stage number by flow |
| CaseID | Single line | FK to Cases list |
| CaseName | Single line | Denormalized case display name |
| StageNumber | Number | Snapshot of stage number at time of transition |
| StageName | Single line | **Text snapshot** — NOT a lookup (preserves audit integrity) |
| PreviousStageNumber | Number | |
| PreviousStageName | Single line | Text snapshot |
| ChangedBy | Person | User who triggered the stage change |
| ChangedDate | Date/Time | Timestamp of stage change |
| TimeInPreviousStageHours | Number | Calculated by flow — duration in previous stage |
| Notes | Multiple lines | Notes recorded at time of transition |
| IsSkipped | Yes/No | True if stage was marked as skipped |

## 5.13 Counters List

**Purpose:** Stores incrementing ID counters used by Power Automate to generate sequential human-readable IDs for cases and leads.

| Column | Type | Notes |
|---|---|---|
| Title | Single line | Counter name — e.g. "BTL", "BRIDGE", "LEAD" |

| PaddingLength | Number | Zero-padding digits — typically 4 |
| LastUpdated | Date/Time | Timestamp of last increment |

**Sample rows:**

| Title | CurrentValue | Prefix | PaddingLength |
|---|---|---|---|
| BTL | 7 | BTL-2026- | 4 |
| BRIDGE | 3 | BRIDGE-2026- | 4 |
| COMMERCIAL | 2 | COMM-2026- | 4 |
| DEV | 1 | DEV-2026- | 4 |
| REFURB | 2 | REFURB-2026- | 4 |
