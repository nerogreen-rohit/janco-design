# 15. Transformation Summary — Before vs After

This section illustrates the transformation from the current manual processes to the new Specialist Lending Platform.

---

## 15.1 Before vs After Overview

| Area | Before (Current State) | After (New Platform) |
|---|---|---|
| **Lead capture** | Email inbox, sticky notes, spreadsheet | Leads list in Staff App — structured, searchable, assignable |
| **Case management** | Excel workbook per case | Cases list with 14-stage lifecycle, SLA tracking, full audit trail |
| **Fact-find** | Word document emailed to customer | SharePoint list item — customer completes online, no email bounce |
| **Document collection** | Email attachments scattered across inbox | Structured document library with 13 subfolders per case |
| **Stage tracking** | Handwritten notes or memory | Stage Configuration list + Stage History — full audit log |
| **Customer communication** | Ad hoc emails | Messages list — threaded, searchable, tied to case |
| **Commission tracking** | Separate Excel spreadsheet | Commissions list — linked to case, automatic totals |
| **Customer progress updates** | Manual phone/email | Automated emails triggered by stage changes |
| **Advisory fee (AdvFee) tracking** | Separate spreadsheet or memory | Commission record per case — IsReceived flag, received date |
| **Compliance / GDPR** | Manual checklist | Compliance Dashboard — TOB status, fact-find completion per case |
| **KPI reporting** | Manual monthly spreadsheet | Live dashboards in Staff App — Pipeline, Performance, Commission, Compliance |
| **Portfolio clients** | Multiple spreadsheet tabs | Multiple case records linked by applicant name/email |
| **Lender management** | Contacts saved in email | Lenders list — BDM details, max LTV, proc fee rate |
| **Referrer management** | Ad hoc tracking | Referrers list — split %, total cases, total fees earned |

---

## 15.2 Process Improvement — Stage-by-Stage

| Stage | Before | After |
|---|---|---|
| **Customer Enquiry** | Lead written on notepad or emailed to self | Lead created in Staff App; assigned to staff member; status tracked |
| **Fact-Find** | Word document emailed back and forth | Customer completes SharePoint list item via portal; advisor reviews in app |
| **Terms of Business** | Printed and posted or Word doc emailed | Issued digitally; future e-sign for instant signature |
| **Product Research** | Manual note-taking | Stage Notes field; internal — not visible to customer |
| **Present Options** | Phone call with no record | Stage Notes + Message to customer — audit trail |
| **DIP** | Submitted manually — no record in tracking system | DIP folder + Stage History entry + case note |
| **Full Application** | Pack emailed manually | "Email Docs to Lender" feature — tracked, logged |
| **Valuation** | Chased manually | SLA reminder flow — automatic chase if overdue |
| **Underwriting** | Chased manually | SLA reminder + overdue flag on dashboard |
| **Loan Offer** | Notified customer manually | Automated customer notification on stage advance |
| **Legals** | Chased manually | Tasks list + SLA monitoring |
| **Completion** | Spreadsheet updated manually | Completion flow — Commission records, customer message, case archived |
| **Post-Completion** | Manual entry in spreadsheet | Commission records confirmed; case status set to Completed; archive |

---

## 15.3 Advisory Fee (AdvFee) — Before vs After

| Aspect | Before | After |
|---|---|---|
| **How tracked** | Separate Excel row | Commission record per case (CommissionType = "Advisory Fee (AdvFee)") |
| **Visibility** | Only system owner could see | All staff can see their cases' commission records |
| **Receipt confirmation** | Manual tick on spreadsheet | IsReceived + ReceivedDate fields; Commission Dashboard |
| **Invoice reference** | Noted in email | InvoiceNumber field on Commission record |
| **Tax year reporting** | Manual annual summary | TaxYear field — filtered in Commission Dashboard |

---

## 15.4 Customer Experience — Before vs After

| Touchpoint | Before | After |
|---|---|---|
| **Fact-find** | Word document by email — confusing, can get lost | Direct link to SharePoint form — simple, mobile-friendly |
| **Document submission** | Email attachments to advisor | Upload to named folder — visible in portal |
| **Progress updates** | Customer has to call/email to ask | Live progress tracker in portal; automated stage notifications |
| **Communication** | Emails only — no central thread | Messages tab — threaded, case-specific, accessible from portal |
| **Knowing what to do next** | Customer waits for advisor to call | "What you need to do" action panel in portal |
| **Contact details** | Advisor's mobile | "Contact your advisor: [email]" always shown in portal footer |

---

## 15.5 Quantified Benefits (Estimated)

| Metric | Before | After | Improvement |
|---|---|---|---|
| Time to complete fact-find | 3–7 days (email delays) | 1–2 days (online, no bounce) | ~60% faster |
| Document chase time per case | 2–4 hours/case | 30 mins/case (automated requests) | ~75% reduction |
| Stage tracking accuracy | ~60% (memory/notes) | 100% (automated stage history) | Full audit trail |
| Commission reconciliation | Monthly manual reconcile | Real-time Commission Dashboard | Near real-time |
| Customer satisfaction | Mixed — slow, confusing | Structured, informed, transparent | Improved NPS |
| Compliance risk | High (manual TOB tracking) | Low (Compliance Dashboard) | Significantly reduced |
