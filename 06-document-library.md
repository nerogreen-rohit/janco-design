# 6. Document Library Structure

**Library name:** Case Documents  
**Location:** Specialist Lending Hub SharePoint site

---

## 6.1 Overview

Each case has a dedicated folder within the Case Documents library. The folder is named using the CaseID (e.g. `BTL-2026-0001`). Within each case folder, there are 13 standardised subfolders.

```
Case Documents/
├── BTL-2026-0001/
│   ├── 01-Identity/
│   ├── 02-Income/
│   ├── 03-Property/
│   ├── 04-Bank-Statements/
│   ├── 05-Fact-Find/
│   ├── 06-Terms-of-Business/
│   ├── 07-Illustrations/
│   ├── 08-DIP/
│   ├── 09-Application/
│   ├── 10-Valuation/
│   ├── 11-Offer/
│   ├── 12-Legal/
│   └── 13-Completion/
├── BTL-2026-0002/
│   └── [same 13 subfolders]
├── BRIDGE-2026-0001/
│   └── [same 13 subfolders]
└── ...
```

---

## 6.2 Subfolder Definitions

| Folder | Contents | Who Uploads |
|---|---|---|
| **01-Identity** | Passport, driving licence, utility bills | Customer (via portal) |
| **02-Income** | Payslips, SA302, company accounts, P60 | Customer (via portal) |
| **03-Property** | Title register, buildings insurance, tenancy agreements, EPC | Customer (via portal) |
| **04-Bank-Statements** | Personal and business bank statements | Customer (via portal) |
| **05-Fact-Find** | Exported/saved fact-find summary PDF (generated by flow) | System (auto) |
| **06-Terms-of-Business** | Signed TOB, privacy notice, GDPR consent | Staff / Customer |
| **07-Illustrations** | ESIS/KFI documents | Staff |
| **08-DIP** | DIP submission and confirmation | Staff |
| **09-Application** | Full application pack, supporting documents for lender | Staff |
| **10-Valuation** | Valuation report, surveyor correspondence | Staff / Lender |
| **11-Offer** | Formal loan offer letter | Staff / Lender |
| **12-Legal** | Solicitor correspondence, completion statement | Staff / Solicitor |
| **13-Completion** | Completion certificate, final statement, archive pack | Staff |

---

## 6.3 Folder Creation

Folders and subfolders are created automatically by Power Automate when a lead is converted to a case. The flow:

1. Creates the top-level case folder named with the CaseID
2. Creates all 13 subfolders within it
3. Updates the Case record with the library folder path URL
4. Applies sharing permissions:
   - **Staff** — Full control of case folder
   - **Customer** — Contribute access to 01-Identity, 02-Income, 03-Property, 04-Bank-Statements only (guest shared link)

---

## 6.4 Email to Lender

The "Email Docs to Lender" automation allows staff to:
1. Select which documents to include from the case folder
2. Trigger a Power Automate flow that compiles selected files
3. Send a formatted email to the lender's submission email address with attachments
4. Log the email event in the Stage History and add a Message record

---

## 6.5 Document Naming Convention

Files uploaded by customers or staff should follow this convention where possible:

`[CaseID]_[DocType]_[ApplicantInitials]_[Date].ext`

**Examples:**
- `BTL-2026-0001_Passport_JD_20260410.pdf`
- `BTL-2026-0001_Payslip_Mar26_JD_20260410.pdf`
- `BTL-2026-0001_SA302_2024-25_JD_20260410.pdf`

> **Note:** Customer uploads via the portal will not always follow this convention. Staff should rename files where necessary before submission to lender.
